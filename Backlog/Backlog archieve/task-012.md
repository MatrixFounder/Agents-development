# Техническое Задание (ТЗ): Переименование артефакта TZ.md и связанных компонентов фреймворка

**Дата:** 15 января 2026  
**Версия ТЗ:** 1.0  
**Ответственный:** Maintainer / Контрибьютор  
**Репозиторий:** https://github.com/MatrixFounder/Agentic-development  
**Цель релиза:** v3.1.0 (патч-релиз, полностью backwards-compatible)  
**Предпосылки:** Текущая версия v3.0.1–v3.1.0. Артефакт `docs/TZ.md` является источником правды по текущей задаче. Существует роль `03_tz_reviewer_prompt.md`. Упоминания "TZ" / "tz" присутствуют в skills, workflows и промптах.

## Цель

Повысить консистентность naming фреймворка за счёт перехода на английские термины, соответствующие общепринятым в open-source агентных системах разработки (Task, Requirements, Specification).  

Это решит проблемы:
- Несоответствие русскоязычного акронима "TZ" (Техническое Задание) англоязычным артефактам (`ARCHITECTURE.md`, `.AGENTS.md` и т.д.).
- Упрощение онбординга для международной аудитории.
- Единообразие терминологии ("task" уже широко используется в промптах: "Current task:", "TASK:").

После реализации:
- Основной артефакт задачи будет называться `docs/TASK.md`.
- Все ссылки и правила обновлены.
- Сохранена совместимость с существующими проектами.

## Новое имя артефакта

- **Основной файл:** `docs/TZ.md` → `docs/TASK.md` (рекомендуется как наиболее короткое и интуитивное).


## Задачи

1. **Переименование файлов**
   - Переименовать `docs/TZ.md` → `docs/TASK.md`.
   - Переименовать `.agent/roles/03_tz_reviewer_prompt.md` → `.agent/roles/03_task_reviewer_prompt.md`.
   - Обновить содержимое нового файла роли (заменить внутренние упоминания "TZ" / "tz" на "TASK" / "task").

2. **Глобальная замена упоминаний "tz" → "task"**
   - Во всех файлах репозитория выполнить поиск/замену:
     - "TZ.md" → "TASK.md" (с учётом регистра).
     - "tz" → "task" в контекстах: имена файлов, пути, переменные, описания (например, "TZ Reviewer" → "Task Reviewer").
     - Исключения: не трогать исторические упоминания в CHANGELOG или комментариях, так как CHANGELOG всегда дописывается.
     - Исключения: не трогать файлы в папке `docs/tasks/`.
   - Ключевые места для проверки и правки:
     - Все skills в `.agent/skills/` (особенно `skill-artifact-management.md`, `skill-core-principles.md`).
     - Все роли в `.agent/roles/` (описания, ACTIVE SKILLS).
     - Workflows в `docs/WORKFLOWS.md` и примеры.
     - Orchestrator код (если есть хардкод путей к TZ.md).
     - README.md, docs/* (все документационные файлы).
     - Примеры задач и архивации.

3. **Обновление правил архивации задач**
   - Текущие правила архивации уже используют формат `task-{num}-{slug}-{number}.md` (как указано в запросе).
   - Уточнить/обновить в `skill-artifact-management.md` и документации:
     - При завершении задачи: архивировать `TASK.md` → `docs/tasks/archived_task_{timestamp}_{slug}.md` (или сохранить текущий формат `task-{num}-{slug}-{number}.md` для консистентности).
     - Удалять/архивировать старые TZ.md только если они остались от предыдущих версий.

4. **Обеспечение backwards-compatibility**

   - Добавить в CHANGELOG и README предупреждение о переименовании и рекомендации мигрировать старые проекты (скопировать новые файлы System/Agents, все системные файлы .cursorrules/GEMINI.md, skills, workflows. Текущий файл TZ.md самостоятельно переименовать в TASK.md).

5. **Обновление документации**
   - В `docs/SKILLS.md`, `docs/WORKFLOWS.md`, README.md обновить все примеры и описания.
   - Добавить раздел в CHANGELOG:
     ```
     ### v3.1.1 — Naming Consistency Update
     - Переименован docs/TZ.md → docs/TASK.md
     - Переименована роль 03_tz_reviewer_prompt.md → 03_task_reviewer_prompt.md
     - Все упоминания "TZ" заменены на "TASK" для консистентности
     ```

## Приёмка

- PR с переименованиями и правками.
- Тестирование:
  - Запуск полного workflow на новой задаче (создание TASK.md, верификация Task Reviewer, архивация).
  - Проверка старого проекта (с TZ.md) — Orchestrator должен корректно читать fallback.
  - Нет регрессий в существующих workflow (Stub-First, VDD, full-robust).
- Минимум 1 approve от maintainer.
- Обновлён CHANGELOG и README.

Это ТЗ полностью покрывает переименование с минимальными рисками. После реализации фреймворк станет ещё более консистентным и дружелюбным к международным контрибьюторам.
