# Необходимо перевести все промпты на английский язык и при этом сохранить исходный вариант

Это хорошая практика для open-source репозитория, особенно если проект имеет русскоязычную аудиторию (как видно по текущему README и промптам). Вот ключевые причины:
## Преимущества сохранения

- История и бэкап: Git уже сохраняет коммиты, но отдельные файлы помогут быстро сравнить старые/новые версии (например, через diff). Полезно для отладки, если английская версия неожиданно поведёт себя хуже в каком-то сценарии.
- Совместимость и выбор: Некоторые пользователи (русскоязычные разработчики) могут предпочитать промпты на русском — определённые модели (Gemini Flash, YandexGPT или даже Claude в русскоязычном контексте) иногда лучше понимают нюансы на родном языке. Сохранение позволит переключаться ("Use Russian agents for this task").
- Инклюзивность: Проект станет доступнее для не-англоязычных контрибьюторов. Можно добавить в README инструкцию: "For Russian prompts, switch to folder ...".
- Тестирование: Легко A/B-тестировать (русские vs английские промпты) на реальных задачах — вдруг русский вариант даёт меньше галлюцинаций в определённых ролях (например, Analyst или Planner).
- Минимальные минусы: Дополнительные файлы не займут много места, а maintenance можно автоматизировать (копировать изменения и переводить только при необходимости).

## Рекомендуемая структура папок
Не смешивать старые и новые в одной папке — это создаст путаницу. Лучше сделать параллельную структуру с явным разделением.

Параллельные папки по языку (рекомендую — самый чистый)

```text
System/
├── Agents/                  # Основная (английская) версия — актуальная
│   ├── 00_agent_development.md
│   ├── 01_orchestrator.md
│   └── ... (все роли)
└── Agents_ru/               # Русская версия (legacy или alternative)
    ├── 00_agent_development.md
    ├── 01_orchestrator.md
    └── ... (копии старых файлов)

```
В .gemini/GEMINI.md или .cursorrules добавить опцию: "If user requests Russian mode, load from System/Agents_ru/".

В README: раздел "Language Versions" с объяснением.
