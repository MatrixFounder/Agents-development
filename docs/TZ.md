# ТЗ: Перевод системных промптов на английский язык и структурирование

## 1. Общее описание
**Задача:** Перевести все системные промпты (агентов) на английский язык, сохранив исходные русские версии в отдельной директории (`System/Agents_ru`).
**Цель:** Обеспечить инклюзивность для англоязычных контрибьюторов, упростить поддержку (единый источник правды), сохранить возможность использования русских промптов для специфических кейсов.

## 2. Список юзер-кейсов

### UC-01: Переключение на английские промпты (Default)
**Актёры:** Система, Разработчик
**Предусловия:** Репозиторий обновлён.
**Основной сценарий:**
1. Разработчик запускает систему (или агент).
2. Система по умолчанию загружает промпты из `System/Agents/` (которые теперь на английском).
3. Агент взаимодействует с пользователем на английском (или следует инструкциям на английском).

### UC-02: Использование русских промптов (Legacy/Alternative)
**Актёры:** Разработчик
**Предусловия:** Разработчик хочет использовать русские версии.
**Основной сценарий:**
1. Разработчик настраивает конфигурацию (например, через `.cursorrules` или `.gemini/GEMINI.md`, или просто копирует файлы).
2. Система загружает промпты из `System/Agents_ru/`.

### UC-03: Структурная миграция
**Актёры:** Разработчик (в ходе реализации)
**Основной сценарий:**
1. Создаётся папка `System/Agents_ru`.
2. Текущее содержимое `System/Agents` копируется в `System/Agents_ru`.
3. Содержимое `System/Agents` переводится на английский язык.

**Критерии приёмки:**
- ✅ Папка `System/Agents_ru` содержит полные копии текущих русских промптов.
- ✅ Папка `System/Agents` содержит английские версии всех файлов.
- ✅ Файл `00_agent_development.md` переведён.
- ✅ Файл `01_orchestrator.md` переведён.
- ✅ Файлы ролей (02-09) переведены.
- ✅ Смысл инструкций сохранён (accuracy > literal translation).

## 3. Ограничения и допущения
- Перевод должен быть качественным, терминология (TZ, Architecture, Stub-First) должна соответствовать международным стандартам или явно определяться.
- Ссылки внутри файлов (например, "Read System/Agents/02_analyst_prompt.md") должны оставаться корректными в английской версии (ссылаться на английские файлы).
- В русской версии ссылки также должны вести на правильные файлы (возможно, стоит оставить их указывающими на `Agents` или перенаправить на `Agents_ru` - *Решение: Русские агенты должны работать изолированно, поэтому ссылки внутри них лучше поправить на `Agents_ru`, если это не сломает путь* -> **Уточнение**: Пока просто бэкап, но лучше, чтобы они были рабочими. Обновим пути imports в русских файлах).
